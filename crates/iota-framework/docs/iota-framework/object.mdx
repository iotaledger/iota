## Struct `ID`

An object ID. This is used to reference Iota Objects.
This is *not* guaranteed to be globally unique--anyone can create an <code><Link to="../iota-framework/object#0x2_object_ID">ID</Link></code> from a <code><Link to="../iota-framework/object#0x2_object_UID">UID</Link></code> or
from an object, and ID's can be freely copied and dropped.
Here, the values are not globally unique because there can be multiple values of type <code><Link to="../iota-framework/object#0x2_object_ID">ID</Link></code>
with the same underlying bytes. For example, <code><Link to="../iota-framework/object#0x2_object_id">object::id</Link>(&obj)</code> can be called as many times
as you want for a given <code>obj</code>, and each <code><Link to="../iota-framework/object#0x2_object_ID">ID</Link></code> value will be identical.


<pre>
<code>
<b>struct</b> <Link to="../iota-framework/object#0x2_object_ID">ID</Link> <b>has</b> <b>copy</b>, drop, store
</code>
</pre>

## Struct `UID`

Globally unique IDs that define an object's ID in storage. Any Iota Object, that is a struct
with the <code>key</code> ability, must have <code>id: <Link to="../iota-framework/object#0x2_object_UID">UID</Link></code> as its first field.
These are globally unique in the sense that no two values of type <code><Link to="../iota-framework/object#0x2_object_UID">UID</Link></code> are ever equal, in
other words for any two values <code>id1: <Link to="../iota-framework/object#0x2_object_UID">UID</Link></code> and <code>id2: <Link to="../iota-framework/object#0x2_object_UID">UID</Link></code>, <code>id1</code> != <code>id2</code>.
This is a privileged type that can only be derived from a <code>TxContext</code>.
<code><Link to="../iota-framework/object#0x2_object_UID">UID</Link></code> doesn't have the <code>drop</code> ability, so deleting a <code><Link to="../iota-framework/object#0x2_object_UID">UID</Link></code> requires a call to <code>delete</code>.


<pre>
<code>
<b>struct</b> <Link to="../iota-framework/object#0x2_object_UID">UID</Link> <b>has</b> store
</code>
</pre>
