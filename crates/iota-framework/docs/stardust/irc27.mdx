</dt>
<dd>
 The media type (MIME) of the asset.

 ## Examples
 - Image files: 
image/jpeg, 
image/png, 
image/gif, etc.
 - Video files: 
video/x-msvideo (avi), 
video/mp4, 
video/mpeg, etc.
 - Audio files: 
audio/mpeg, 
audio/wav, etc.
 - 3D Assets: 
model/obj, 
model/u3d, etc.
 - Documents: 
application/pdf, 
text/plain, etc.
</dd>
<dt>

uri: <Link to="../iota-framework/url#0x2_url_Url">url::Url</Link>
</dt>
<dd>
 URL pointing to the NFT file location.
</dd>
<dt>

name: <Link to="../move-stdlib/string#0x1_string_String">string::String</Link>
</dt>
<dd>
 Alphanumeric text string defining the human identifiable name for the NFT.
</dd>
<dt>

collection_name: <Link to="../move-stdlib/option#0x1_option_Option">option::Option</Link>&lt;<Link to="../move-stdlib/string#0x1_string_String">string::String</Link>&gt;
</dt>
<dd>
 The human-readable collection name of the NFT.
</dd>
<dt>

royalties: <Link to="../iota-framework/vec_map#0x2_vec_map_VecMap">vec_map::VecMap</Link>&lt;<b>address</b>, <Link to="../move-stdlib/fixed_point32#0x1_fixed_point32_FixedPoint32">fixed_point32::FixedPoint32</Link>&gt;
</dt>
<dd>
 Royalty payment addresses mapped to the payout percentage.
 Contains a hash of the 32 bytes parsed from the BECH32 encoded IOTA address in the metadata, it is a legacy address.
 Royalties are not supported by the protocol and needed to be processed by an integrator.
</dd>
<dt>

issuer_name: <Link to="../move-stdlib/option#0x1_option_Option">option::Option</Link>&lt;<Link to="../move-stdlib/string#0x1_string_String">string::String</Link>&gt;
</dt>
<dd>
 The human-readable name of the NFT creator.
</dd>
<dt>

description: <Link to="../move-stdlib/option#0x1_option_Option">option::Option</Link>&lt;<Link to="../move-stdlib/string#0x1_string_String">string::String</Link>&gt;
</dt>
<dd>
 The human-readable description of the NFT.
</dd>
<dt>

attributes: <Link to="../iota-framework/vec_map#0x2_vec_map_VecMap">vec_map::VecMap</Link>&lt;<Link to="../move-stdlib/string#0x1_string_String">string::String</Link>, <Link to="../move-stdlib/string#0x1_string_String">string::String</Link>&gt;

 Additional attributes which follow [OpenSea Metadata standards](https://docs.opensea.io/docs/metadata-standards).
